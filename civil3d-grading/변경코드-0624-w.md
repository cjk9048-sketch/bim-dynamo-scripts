## 1) GradingEngine.cs — ①통합지표면을 '구성측(diff>0)'만 생성 (버퍼 절벽 제거)

**위치**: `GradingEngine.Run()` 격자 스캔 루프, 외부(비탈) 셀 처리부.

**핵심**: 예전엔 daylight 너머 1단 '버퍼'까지 표면 점에 넣고 나중에 잘랐는데, 그 버퍼가 절토(위로)·성토(아래로)
전환부에서 맞물려 절벽(=닫는 면 이상 채움)을 만들었다. → **버퍼를 표면에 안 넣고 구성측(diff>0)만** 넣는다.

```csharp
// (앞부분 동일) … d, proposed, diff 계산 후:
result.DiffPoints.Add(new Point3(x, y, proposed - gCell)); // 차이표면(daylight 양쪽 버퍼 포함) — daylight 폴백 전용
dist[ix, iy] = d;
if (diff > 0)
{
    // ★통합지표면은 '구성측(diff>0)'만 — daylight 너머 1단 버퍼를 표면에 넣지 않는다.
    //   버퍼를 넣으면 절토(위로 105+h)·성토(아래로 105−h) 슬리버가 절↔성 전환부에서 맞물려
    //   '닫는 면이 이상하게 채워지는' 절벽(2h)을 만든다(JACK 반복 지적 문제). 절토·성토가 각자
    //   daylight까지만 닫히고, 사이(원지반≈계획고 전환구간)는 비워 둔다 → 외곽 폐합선으로 트림.
    result.Points.Add(new Point3(x, y, proposed));   // ← 예전엔 이 줄이 if 밖(버퍼 포함)이었음
    cons[ix, iy] = true;
    zField[ix, iy] = proposed;
    result.SlopeCellCount++;
    AccumulateVolume(result, ground, x, y, proposed, cellArea);
}
```

> daylight 폐합선 생성부(`cons` 마스크 경계 추적)는 그대로다 — `cons`는 위에서 그대로 세팅되므로
> daylight/토량 결과는 불변. 바뀐 것은 '표면 점' 집합뿐.

## 2) CreateGradingCommand.cs — ②외곽 폐합선만 + ③경계 트림

### 2-a. daylight 결정: 격자 cons-마스크 경계를 1순위, 면적 최대 1개만

```csharp
// ── tr2: daylight 결정 — ① 격자 cons-마스크 경계(오목 L자/혼합 절성토에 위상 견고: 내부 절↔성
//         전환선은 cons '안쪽'이라 안 잡힘 → 내부 가짜선 없음) 우선, 면적 최대 1개만 사용.
//         ② 비면 기하 레이마칭 ③ 그래도 비면 차이표면 0등고선. ──
var dayLoops = KeepLargestLoop(result.DaylightLoops);
string dayMethod = "격자(외곽1선)";
if (dayLoops.Count == 0 && geoDay != null && geoDay.Count > 0)
{
    dayLoops = geoDay;
    dayMethod = "기하 폴백";
}
if (dayLoops.Count == 0)
{
    using (Transaction tr = db.TransactionManager.StartTransaction())
    {
        dayLoops = SurfaceBuilder.ExtractDaylightContour(db, tr, result, gradeSurfId, boundary, p.CellSize);
        tr.Commit();
    }
    dayMethod = dayLoops.Count > 0 ? "차이표면 폴백" : "⚠ daylight 없음";
}
```

### 2-b. 외곽 1개만 남기는 헬퍼 (내부 닫힘 전부 제외)

```csharp
/// <summary>여러 daylight 루프 중 면적이 가장 큰 1개만 남긴다(외곽). 내부 봉우리 구멍·잔루프 제외 — 15·17차 방침.</summary>
private static List<List<Point3>> KeepLargestLoop(IReadOnlyList<List<Point3>> loops)
{
    var outp = new List<List<Point3>>();
    if (loops == null || loops.Count == 0) return outp;
    int best = -1; double bestA = -1;
    for (int i = 0; i < loops.Count; i++)
    {
        var l = loops[i];
        if (l == null || l.Count < 3) continue;
        double a = 0; int n = l.Count;
        for (int j = 0, k = n - 1; j < n; k = j++) a += l[k].X * l[j].Y - l[j].X * l[k].Y;
        a = Math.Abs(a) * 0.5;
        if (a > bestA) { bestA = a; best = i; }
    }
    if (best >= 0) outp.Add(loops[best]);
    return outp;
}
```

### 2-c. ③경계로 자르기 + 빗살 제거 (addVertices 끔)

```csharp
// 통합지표면은 구성측(daylight 직전)까지만이라 가장자리 삼각형이 작다 → daylight 점을 정점으로
// 따로 박지 않는다(addVertices:false). 박으면 cons 셀점과 겹쳐 얇은 슬리버(빗살)가 생김. 파괴식
// 외곽 경계가 알아서 그 선을 따라 삼각형을 잘라 매끈한 가장자리를 만든다.
trimmed = SurfaceBuilder.TrimToGroundIntersection(tr, gradeSurfId, dayLoops, addVertices: false);
```

> `TrimToGroundIntersection`은 `grade.BoundariesDefinition.AddBoundaries(pc, 1.0, Outer, false)` —
> 즉 표면의 **경계 기능**으로 외곽선 밖 삼각형을 파괴식으로 잘라낸다(=JACK '경계로 자르기').
