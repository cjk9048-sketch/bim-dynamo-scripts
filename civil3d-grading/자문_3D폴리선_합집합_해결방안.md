# 3D 폴리선 합집합 및 위상 에러 해결 자문

## 개요
현재 도출하신 **`절토 경계 = 절토교선루프들 ∪ (계획폴리곤 − Union(성토교선루프들))`** 로직은 '계획직선 sticking' 현상을 해결하기 위한 수학적으로 완벽하고 정확한 접근입니다. 절토와 성토의 전이 구간에서 발생하는 문제를 반대편 교선을 빼는(Difference) 방식으로 우회한 것은 매우 훌륭한 해결책입니다.

우려하시는 지점들에 대해 NTS(NetTopologySuite) 기반의 해결책과 Civil 3D API 특성을 고려한 자문 내용입니다.

## 1. 전이선 좌표 일치 문제 (미세 틈새/슬리버 봉합)
각각 다른 TIN에서 도출된 절토 루프와 성토 루프의 전이선은 이론상 같은 선이어야 하지만, 부동소수점 연산의 한계로 수 mm~cm의 어긋남이 반드시 발생합니다. 이 상태로 Union을 하면 눈에 보이지 않는 미세한 구멍이나 자기 교차 링(Self-intersecting ring)이 생성됩니다.

**표준 기법:** NTS의 `GeometrySnapper`를 사용하여 Union 전에 두 기하구조의 정점을 강제로 일치시키는 것이 가장 견고합니다.

```csharp
// planPart와 ownGs(절토 루프들)를 병합하기 직전, 허용 오차 내의 정점을 스냅
double snapTolerance = 0.01; // 1cm (Civil 3D 지형에서 합리적인 허용치)
var combinedOwnGs = UnaryUnionOp.Union(ownGs);

// ownGs의 외곽선을 planPart의 정점에 스냅 (또는 그 반대)
var snappedOwn = GeometrySnapper.SnapTo(combinedOwnGs, planPart, snapTolerance);
var snappedPlanPart = GeometrySnapper.SnapTo(planPart, combinedOwnGs, snapTolerance);

// 이후 Union 수행
var u = snappedOwn.Union(snappedPlanPart);
```

## 2. PrecisionModel(고정 정밀도 모델)의 효과와 부작용
NTS 2.x부터 도입된 `OverlayNG`는 매우 강력하며, 고정 정밀도 모델(`PrecisionModel`)을 적용하면 위에서 언급한 미세 어긋남 자동 봉합에 큰 도움이 됩니다.

*   **효과:** `new PrecisionModel(1000)`(1mm 격자)을 적용하여 기하 연산을 수행하면, 1mm 이내에 있는 정점들이 동일한 좌표로 취급되어 슬리버가 원천적으로 차단됩니다.
*   **부작용 (위상 붕괴):** 정점이 이동하면서 1mm 폭보다 좁은 정상적인 뾰족한 면이 '선'으로 붕괴(Collapse)될 수 있습니다. 하지만 토공 설계 척도(스케일)에서 1mm 이하의 뾰족한 면은 의미가 없으며, 오히려 Civil 3D Boundary에서 문제를 일으키는 원인이 되므로 이러한 붕괴는 우리에게 이로운 부작용입니다.

## 3. Civil 3D가 수용하는 외곽선 후처리 권장 순서
Civil 3D의 `AddBoundaries(Outer)`는 자기 교차(Bowtie)나 닫히지 않은 선형에 매우 민감하여 치명적인 오류를 뿜어냅니다. 도출된 합집합 결과를 Civil 3D로 넘기기 전 다음 순서로 정제하는 것을 권장합니다.

1.  **2D 위상 정리:** `u = u.Buffer(0)` (기존 방식 유지)
2.  **Exterior Ring 추출:** 합집합 결과가 다중 폴리곤(`MultiPolygon`)이거나 내부 구멍(`Holes`)을 포함할 수 있습니다. Outer Boundary 1개가 필요하므로, 면적이 가장 큰 폴리곤의 `ExteriorRing`만 추출합니다.
3.  **중복 정점 제거:** NTS 연산 후 연속된 정점이 동일한 좌표(2D 기준)를 갖는 경우가 생깁니다. 이를 Civil 3D 폴리선으로 만들면 "길이가 0인 선분"이 되어 Boundary 추가 시 실패할 수 있으므로, 배열 변환 전 연속 중복 좌표를 필터링합니다.
4.  **Z값 역투영 (현재 로직 유지):** 2D 외곽선 정점이 모두 확정된 *후에* 원본 3D 폴리선 중 최근접 선분에서 Z를 선형 보간하여 3D 폴리선으로 조립합니다.

## 4. 대안적 접근 (Polygonizer 네트워크)
현재의 Boolean 연산 방식(`절토 ∪ (계획 - 성토)`)이 면적 검증까지 마쳤다면 성능 면에서 가장 우수하므로 유지하는 것을 추천합니다. 

다만, 향후 전이선 양끝이 계획 경계에 닿지 않는 등 위상 구조가 더 복잡해져 Boolean이 계속 깨진다면, **노드 네트워크 기반 Polygonizer 방식**이 최종 방어선이 됩니다.

1.  계획선, 절토교선, 성토교선을 모두 분해하여 선분(LineString) 모음으로 만듭니다.
2.  NTS의 `UnaryUnion`을 선분 모음에 적용하여 교차점마다 노드(Node)를 생성하고 선을 쪼갭니다(Noding).
3.  이 쪼개진 선들을 `Polygonizer`에 넣어 생성 가능한 모든 닫힌 면(Face)을 만듭니다.
4.  각 면의 중심점(Centroid)이 원본 절토 구역 안에 있는지, 계획 구역 안에 있는지 판별(Point-in-Polygon)하여 필요한 면만 수집한 뒤 다시 합칩니다.

이 방식은 Civil 3D가 내부적으로 사이트(Site)와 구획(Parcel)을 나누는 방식과 동일하여 위상적으로는 가장 무결하지만, 연산 부하가 크고 중심점 판별 로직을 섬세하게 짜야 합니다. 따라서 현재의 로직에 `GeometrySnapper`와 `PrecisionModel(1000)`을 결합하여 슬리버를 통제하는 방향으로 먼저 돌파해 보시기를 권장합니다.
