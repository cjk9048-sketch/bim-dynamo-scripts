## 0-CC. (2026-06-30) — ★daylight '영역 기반' 재설계 (11시 누락·옹벽 톱니)

**경위(0626~0630)**: 0626-p까지 daylight = 교선(가상면↔원지반 marching) 방식. 문제 연쇄 발견·해결:
- **섬/슬릿**: 절토·성토 daylight가 폭0 슬릿(self-touching)으로 겹쳐 섬 생성 → `RemovePinchSlits`(겹침점에서 작은 슬릿 컷) + `MergeDaylightOutlines`(그리기용: 절토∪성토∪계획경계 union해 바깥 외곽만) + 8자 폴백. **(커밋 05731ee)**
- **11시 성토 누락**: 비탈이 측량 원지반 밖에서 만나(교선 없음) daylight가 안 닫힘. → **daylight 영역 기반 재설계**: 원지반 삼각형마다 `Diff×sign>0` 영역(성토=가상면이 원지반보다 높은 곳, 절토=낮은 곳)을 `PositivePart`로 잘라 `UnaryUnion` → 바깥 외곽선이 daylight. `sign`(cut=-1/fill=+1), Z=원지반 보간. 영역 경계는 측량 끝에서도 닫혀 11시 해결. **(커밋 cf41eca) 완만 구배·11시 정상 검증됨.**
- **옹벽(구배<0.1, 수직)**: daylight가 톱니. 원인 = 수직 영향 띠(폭 수십cm)가 원지반 삼각망(수m)보다 좁아 거칠게 근사. 흰선(Civil '지표면 최소거리')처럼 **두 표면을 정밀 교차**해야 함(자문 #11 = 면-면 평면식 교차).
- **해결(미커밋·런타임 검증 대기)**: 면-면 교선(`RawTriangleIntersectionFinder`)은 폐합 약점(교선 조각→가상면 300개 폭증)으로 폐기. → **영역 기반을 '양쪽 표면 삼각망'으로 확장**: `DaylightExtractor`가 원지반+가상면 tris를 합쳐 marching, `DiffVG(p)=virtualSurf(p)−groundSurf(p)`로 양쪽 직접 샘플. 옹벽(가상면 삼각형이 디테일 주체)도 정밀 + union이라 폐합 자동 + 11시 유지. 자문 #11과 같은 효과(두 표면 정밀)지만 폐합이 보장됨.

**핵심 파일**: `Core/GradingGeometry.cs`(ExtractDaylightFromMesh=영역기반, PositivePart, DiffVG, RemovePinchSlits, MergeDaylightOutlines), `Civil/DaylightExtractor.cs`(양쪽 tris 수집·bbox클립), `Commands/CreateGradingCommand.cs`(sign 전달).

**다음**: 옹벽(0.05)·완만(1:1.5) 런타임 검증 → 가상면 개수 정상화·daylight 매끈·생성 시간 확인. OK면 커밋.

## 0-AC. (2026-06-24, `0624-ac/ad·Ray단면`) — 정점방식(각짐) → Hybrid로 대체

**외부 AI 자문(`참고자료/grading3.txt`)**: ①NTS Buffer오프셋+클립이 '벽'의 주범(2D로 밀고 칼로 자르니 공중에서 끊겨 벽) → **클립 폐기**. ②정답 = **Ray Casting(단면)**: 경계 1m 쪼개 각 점에서 바깥 법선으로 단면(사면+소단) 쌓되 **원지반과 만나면 종료** → 전환부 영점에서 **단 0 자연수렴(pinch-out)→벽 없음**. ③daylight=**Breakline+Outer 둘 다**(경계로만 쓰면 보간 틀어짐). ④수직단차 금지(소단 미세물매 0.01). ⑤NTS는 daylight 꼬임(bow-tie)만 Buffer(0)로. 방향(NTS+TIN) 맞다 확인.

**구현(0624-ac, Ray-casting 전면 재작성)**:

- command: Ray-casting 1표면 `정지면_DHGrade` + 초록 daylight선. 격자/Buffer클립 미사용.
- 빌드0·테스트58·설치(13:00). 팝업 `[버전 0624-ac·Ray단면(격자/클립폐기)]`.

JACK 요청: **DHGRADE에서 초록 형상선 그리기 제거**(노리선/SHP 때만 활용). CreateGradingCommand에서 DrawPolylines 제거(EraseOnLayer는 남겨 재실행 시 이전 형상선 정리), 팝업 형상선 줄·lineCount 제거. (형상선을 노리선/SHP 출력으로 넣는 건 추후.)
     ② **평지 가장자리(계획 폴리곤)가 경계 안쪽에 들쭉날쭉** — `BuildGridSurface`가 경계를 일부러 안 넣어(주석) 표면 외곽이 격자점만으로 형성 → 변마다 0~1셀 안쪽. **해결: 경계를 0.5m 간격 '점'으로 표면 정점에 추가**(브레이크라인 아님 → 교차 이벤트 없음, 과거 폭주는 브레이크라인일 때만). 평지 가장자리가 입력 경계에 정확히 닿음.
   - **남은 것**: DHGRADE에서 "경계 밖으로 약간 나간 형상선"은 이제 형상선 미표시로 무관. 정지면(표면) 외곽의 daylight 계단(아래 5번)은 미해결.
   - **7차(0619-h·노리선형상선이관)**: 형상선 제거 후 발견 — DHSLOPE는 빗살(틱)만 그리고 소단/사면 모서리선은 "DHGRADE의 DH-형상선 참조"로 **DHGRADE에 의존**하고 있었음(주석). DHGRADE에서 형상선을 빼니 **아무도 소단/사면 모서리선을 안 그림**. JACK 의도=노리선이 소단·사면선 표시. **해결: DHSLOPE가 `GradingEngine.Run` 호출해 `BenchLoops`를 DH-형상선(초록)에 그림(빗살+형상선 함께, 재실행 시 삭제 후 재작성)**. DHGRADE는 DH-형상선 레이어를 더 이상 건드리지 않음(노리선이 소유 → DHGRADE 재실행이 노리선 형상선을 지우는 충돌 방지). SHP(ExportShpCommand)는 `ExtractAreaPolygons` 자체 면폴리곤(소단/전체부지/계획) — 형상선과 무관, 그대로 유효.

   - **8차(i·노리선기하재작성)**: JACK 노리선 5문제 — ①부지경계 계단 ②노리선이 위로 떡지게 떠오름 ③경계-소단선 안맞고 넘어감 ④노리선이 형상선에 안 닿음 ⑤노리선 길이 일정(긴선=사면전체, 짧은선=절반이어야).
     진단(3D 스샷 160732): 노리선이 표면 위로 텐트처럼 **떠 있음** — 노리선이 **옛 격자(TraceBoundary)** 기반이고 빗살이 **양끝 같은 z(평평)**라 사면 위로 뜸. 형상선(기하)과 따로 놀아 안 닿음(④), 격자라 계단(①③).
     **해결: `ExtractSlopeTicks`를 기하 오프셋 위에 재작성** — `EdgeFrames`(변 방향/법선/볼록) + `BuildOffsetRingN`(거리 d의 오프셋 링을 점+바깥법선으로; 볼록=원호, 오목=마이터, 형상선과 동일 기하) → 빗살 밑동이 형상선과 같은 등고선에 얹힘(④), 격자 계단 제거(①③ 일부). 빗살이 사면 경사 따라 **하강(끝점 z=zTop−H·L/faceRun)** → 안 뜸(②). 길이 긴선=faceRun·짧은선=½, 5m마다 긴선(⑤). 회귀 테스트 `SlopeTicks_SlopeDownOutward`(하강·바깥) + `SlopeTicks_SlopeDownNotFloating_LongAndShort`(길이2종). 55개 통과. ※ 정지면(표면) 외곽 daylight 계단(①의 표면쪽)은 여전히 격자 트림 → 아래 5번에서 기하 daylight로 해결 예정.

   - **9차(k·기하daylight)**: JACK 확정 — "계단"=**회색 정지면(TIN) 외곽**(격자 트림), "노리선 없는 곳"=**맨 바깥(땅 닿는) 사면**. 진단: 입력경계 깨끗(8각 직각), 안쪽 형상선 깨끗, 바깥 단만 물결 daylight로 파편화(z=112+ 루프 4~5개). 첫사면 노리선은 재현상 생성됨(zTop별 분포 확인) → 누락은 '맨 바깥 사면'.
     **해결(외곽 계단): `ExtractGeometricDaylight` 추가** — 경계 각 점에서 바깥 법선으로 ray-march해 정지제안고=원지반 만나는 toe를 찾아 매끈한 외곽 폐합선 생성(격자 안 거침). Run에서 라운드 모드면 격자 외곽 daylight를 이걸로 교체(안쪽 봉우리 구멍은 격자 유지, 면적<격자외곽*0.5거나 절반미만 도달시 폴백). 표면 트림(TrimToGroundIntersection)이 이 매끈한 외곽을 써서 **표면 외곽 계단 제거**. 테스트 `GeometricDaylight_SmoothEnclosing`(닫힘·감쌈·회전각≤2π·1.3). 57개 통과. ※ DHGRADE에서 확인. 맨바깥 사면 노리선(#3)·overshoot(#1)은 스샷 보고 후속.

   - **10차(l·daylight스무딩)**: 9차 기하 ray-march daylight는 격자 계단은 없앴으나 **너무 러프**(경계 법선 직선 march라 실제 toe 못 따라가고 헐겁게 부풀어 슬리버 발생). JACK: "원지반 닿는 부분 잘 못 따라가고 너무 채워버림". **방향 전환: 부정확한 기하 ray-march 폐기 → 정확한 격자 daylight(셀마다 실제 지반 비교)를 그대로 쓰되 `SmoothChain`(이동평균 6회)으로 격자 계단만 펴기.** 점을 촘촘히 유지(simplifyTol=cell*0.5) 후 스무딩 → 실제 toe에 ~0.5m로 붙으면서 매끈. ExtractGeometricDaylight는 미사용(테스트만 유지). 테스트 `RunDaylight_SmoothedNotStaircase`(감쌈+회전각≤2π·1.6). 58개 통과. DHGRADE에서 확인.

   - **11차(m·daylight정점)**: 10차(이동평균)에도 표면 외곽이 다시 계단. JACK 핵심 지적: "daylight(원지반과 닿는 선)는 격자가 아닐 텐데 왜 격자로?" → **근본원인 규명**: 파괴식 트림(`AddBoundaries(...,Outer,false)`)은 daylight 폴리곤 **바깥 삼각형을 통째로 숨길 뿐 그 선으로 자르지 않음** → 남는 가장자리는 격자 삼각형 모서리(계단). daylight '선'을 매끈하게 해도 표면 가장자리엔 반영 안 됨. **해결: `BuildGridSurface`에서 daylight 선의 점들을 표면 정점으로 추가**(평지 경계를 정점으로 박아 고친 것과 동일 원리) → 삼각망이 daylight 선을 따라가 가장자리 매끈. DHGRADE에서 확인. ※ 만약 잔여 계단이면 다음 단계는 트림을 비파괴 브레이크라인으로(이벤트 감수) 또는 격자를 daylight에서 끊기.

5. **★ 경계 daylight — 격자 daylight(정확)+스무딩+표면 정점화(11차). 노리선 맨바깥 사면(#3)·overshoot(#1) 후속.**
   - 현재: `DaylightLoops`가 격자 cons 마스크의 `TraceBoundary` → 축정렬 계단. TIN 외곽 트림도 이걸 써서 표면 외곽도 계단(기존 미해결 #1·#2).
   - 해결안: 둘레를 촘촘히 샘플 → 각 점에서 바깥 법선으로 **원지반과 정지제안고가 만나는 지점까지 ray-march**(SlopeHatch의 ClipTickAtDaylight과 동일 원리) → 매끈한 daylight 폴리라인. 오목 겹침은 참거리 클립으로 정리. 이 매끈한 점들을 ① 화면 daylight 선 ② TIN 트림 경계로 공유하면 #1·#2 동시 해결. 영향범위: TrimToGroundIntersection·SHP export라 형상선 누락 확정 후 별도 단계로.

## 3. 네이티브 Grading API 조사 결론 (재시도 금지 — 시간낭비)

- 그러나 **GradingGroup 타입 없음**, Create 시그니처가 베이스 `Create(Type, IntPtr,…)`와 섞여 컴파일러로
  못 끄집어냄. 문서·IntelliSense 없이 블라인드로 알아내기 비현실적.
- JACK 리서치도 확인: **그레이딩 객체 생성 API는 사실상 막힘**(레거시 불안정). 우회는 ① 코리더 ② TinSurface 직접 ③ Dynamo.
- `FeatureLine.Create`/`Site.Create` 됨. `FeatureLine.Offset` 없음. AutoCAD `Curve.GetOffsetCurves` 됨(단 자기교차 정리 안 함).
- **결정**: ②(TinSurface 직접) 중 단일값 방식으로 간다(현행). 코리더는 JACK이 처음 거부한 보타이 재현 위험으로 보류.

## 4. 명령 / 설정 / 레이어
- `DHGRADESET` 팝업: 단높이(5)·소단폭(1)·절토구배(1:1.0)·성토구배(1:1.5)·**격자해상도(0.5, 소규모는 0.25~0.1)**.
  구배 0=수직 옹벽(MinSlope 0.01로 클램프).
- `DHGRADE`: 정지면 + `DH-형상선`(초록) 형상선.
- `DHSLOPE`: 노리선(`DH-사면노리` 주황, 짧은선1m/긴선5m, 사면만) + 경계선(`DH-정지경계` 청록). 재실행 시 기존 삭제.
- 토공량: 팝업 숫자. 체적 대시보드 자동등록은 API 막힘(수동 추가만).
