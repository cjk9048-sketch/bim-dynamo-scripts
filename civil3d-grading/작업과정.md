# Civil3D 계단식 정지(절성토) 플러그인 `DH.Grading` — 작업기록 (이어서 작업용)

> **대상**: Civil3D 2026 / .NET 8. **사용자**: JACK(상하수도, 비개발자). **호칭은 "JACK"**.
> **용도**: 배수지·정수장 같은 **소규모** 부지 정지(도로·철도 대규모 아님).
> **목표**: 계획 폴리곤에서 5m 비탈+1m 소단 반복 절성토를 자동 생성 + 깨끗한 형상선 추출 + 토공량.
> **이 PC에서 빌드·설치·테스트 가능**(Civil3D 2026 설치됨). JACK이 런타임 테스트 → 스샷 피드백 반복 중.

## ★ v1.0 (2026-07-06) — 현재 완성 상태 요약 (커밋 기준)

**리본 "DH 정지" 탭 — 버튼 4개**(각기 다른 런타임 글리프 아이콘):
1. **정지 설정(DHGRADESET)** — 단높이·소단폭·구배·격자. 구배 0.05 이하 입력은 자동 0.05로(Civil3D TIN 오류 방지).
2. **정지면 생성(DHGRADE)** — 계획폴리곤+원지반 → 계단식 절/성토 TIN(정지면_DH). 결과를 도면 번들(NOD XRecord)에 저장.
   순수 절토/성토 부지도 정상(반대쪽 억지생성 제거, 도넛은 절·성 둘 다일 때만 → 계획부지 구멍 방지).
3. **노리선(DHNORI)** — 번들 기반, 재선택 없이 사면선·소단선·노리선 틱 작도(경계 클립).
4. **INFRAWORKS(DHINFRA)** — SHP 일괄 내보내기(폴더 선택): 계획면·순절토/성토·면폴리곤(종류/단/표고)·**옹벽선**.
   ※ 지표교선(DHXSEC) 버튼은 제거(명령은 존치).

**옹벽선(INFRAWORKS, 구배 ≤ 0.1 수직옹벽일 때만)** — JACK ARRAY 경로용, 확정 규칙:
- 방향: 사면 **상단(crest)** 선(블록을 경로선 아래에 매다는 방식).
- **1원칙**: 소단(수평, Z 일정)을 만나면 사면선은 거기서 끝(소단 위로 안 그림). 3단이면 단별 옹벽선. → 단 경계 Z-분할 + 수평(소단) 꼬리 제거.
- **절토 지표절단**: 절토 옹벽 상단은 지형 따라 사선으로 잘림 → 그 daylight 사선도 포함(성토는 상단=계획고라 직선).
- **조인**: 각 단의 직선부(crest)+지표절단(daylight)을 한 옹벽선으로 이어붙임(ARRAY 가능하게).
- 좌표계 .prj = EPSG 5186(중부원점, 설정 `ExportEpsg`로 5185/5187/5188·구원점 5180~5184 가능). DBF UTF-8.

- ✅ **"!"(최신아님)**: daylight 경계를 임시 폴리라인으로 추가 후 삭제 → 참조 끊김이 원인. **점목록
  (Point3dCollection)으로 직접 경계 추가**로 변경(삭제 엔티티 없음). → 해결 예상, JACK 재확인 대기.
- ✅ **이벤트 "오류 2180개"**: 실제 오류 1개, 나머지 2179개는 **"중복점 무시"(INFO, 무해)**. 브레이크라인 점
  **전역 중복제거**로 로그 도배 억제. → JACK 재확인 대기.
- ⚠️ **경계 직선 채움(미해결 핵심)**: daylight가 격자 안에서 안 닿는 쪽에서 직선으로 메워짐. **마진(daylight
  도달 범위)을 needed+5로 키움**. 그래도 남으면: 그 직선이 격자 가장자리인지 확인 필요(JACK에게 확대 스샷+
  그쪽 지형이 계속 떨어지는지 질문해둠). **다음 세션에서 우선 확인할 것.**

### 현행 DHGRADE 흐름 (CreateGradingCommand)
- tr1: 경계 폴리곤+원지반 선택 → `GradingEngine.Run` → `BuildFromBenchLoops`(형상선=브레이크라인) →
  형상선을 `DH-형상선` 레이어(초록)에 폴리라인으로 그림 → commit.
- tr2: `TrimToGroundIntersection`(daylight 폐합선들로 Outer/Hide 경계 트림 + 토량지표면) → commit.
- 팝업에 형상선 개수·토공량·버전.

### GradingEngine.Run 요지 (단일값)
- 격자 스캔: 내부=계획면(designZ), 외부=거리d로 BenchProfile 계단Z(daylight 너머 1단 버퍼까지).
- 셀별 `cons`(구성측=diff>0), `dist`(경계거리) 저장.
- `TraceBoundary(mask)`: 마스크 영역 경계를 방향성 에지 체인으로 추적 → 닫힌 루프(외곽+구멍).
- **daylight**: cons 마스크 경계 → DaylightLoops(점 Z=원지반 표고=toe).
- **형상선**: 각 break 거리 d_e(비탈끝·소단끝)의 `cons && dist<=d_e` 마스크 경계 → BenchLoops(점 Z=단표고).
- v1: 부지 전체 한 방향(중심부 designZ vs 원지반). **혼합 절성토 미구현(다음 단계)**.

## 3. 네이티브 Grading API 조사 결론 (재시도 금지 — 시간낭비)
- `Autodesk.Civil.DatabaseServices.Grading`, `Styles.GradingCriteria` **타입은 존재**, `Grading.Create`도 존재.
- 그러나 **GradingGroup 타입 없음**, Create 시그니처가 베이스 `Create(Type, IntPtr,…)`와 섞여 컴파일러로
  못 끄집어냄. 문서·IntelliSense 없이 블라인드로 알아내기 비현실적.
- JACK 리서치도 확인: **그레이딩 객체 생성 API는 사실상 막힘**(레거시 불안정). 우회는 ① 코리더 ② TinSurface 직접 ③ Dynamo.
- `FeatureLine.Create`/`Site.Create` 됨. `FeatureLine.Offset` 없음. AutoCAD `Curve.GetOffsetCurves` 됨(단 자기교차 정리 안 함).
- **결정**: ②(TinSurface 직접) 중 단일값 방식으로 간다(현행). 코리더는 JACK이 처음 거부한 보타이 재현 위험으로 보류.

## 4. 명령 / 설정 / 레이어
- `DHGRADESET` 팝업: 단높이(5)·소단폭(1)·절토구배(1:1.0)·성토구배(1:1.5)·**격자해상도(0.5, 소규모는 0.25~0.1)**.
  구배 0=수직 옹벽(MinSlope 0.01로 클램프).
- `DHGRADE`: 정지면 + `DH-형상선`(초록) 형상선.
- `DHSLOPE`: 노리선(`DH-사면노리` 주황, 짧은선1m/긴선5m, 사면만) + 경계선(`DH-정지경계` 청록). 재실행 시 기존 삭제.
- 토공량: 팝업 숫자. 체적 대시보드 자동등록은 API 막힘(수동 추가만).
