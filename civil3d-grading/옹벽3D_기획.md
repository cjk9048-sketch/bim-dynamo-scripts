# 옹벽 3D (보강토 블록 모델링) — 확정 기본계획 (2026-07-20 인터뷰 완료)

> JACK 기획서(그리드 필터링 + Sweep 캡) + 인터뷰 확정사항. **코딩 전 계획 단계** — 구현은 별도 승인 후.

## 1. 위치·산출물 (인터뷰 확정)
- **DHINFRA에 포함** — 별도 버튼 아님. 기존 옹벽선 SHP를 뽑던 이유가 궁극적으로 이것.
  옹벽 3D가 완성·검증되면 **옹벽선_*.shp 출력은 제거**(검증 전까지는 병행 유지).
- 산출: 선택 폴더에 **`옹벽3D.dwg` 별도 파일만**(현재 도면은 건드리지 않음 — 도면 무거워짐 방지).
  InfraWorks가 이 DWG를 불러옴.
- **`옹벽물량.csv` 동시 출력**: 벽별·단별 블록 개수(정수), 캡 콘크리트 길이·체적. 엑셀 자동화 대비.

## 2. 핵심 방식 (기획서 채택 + 확정)
- **그리드 필터링**: 옹벽 정렬선(기존 옹벽선 v2의 벽 정렬선)을 따라 수평 조적조(엇갈림) 가상 그리드를
  만들고, 각 블록 자리의 Z가 **옹벽선(상단 커팅라인) 아래일 때만** 블록 생성. 블록은 항상 수평 —
  사선 구간은 자연히 계단식(실제 시공과 동일, 빈틈 없음). 경로 따라 기울여 배열하는 방식 폐기.
- **벽 바닥(하단)**: 정지설정 **단높이만큼만** — 각 단의 벽은 단 바닥레벨(수평)부터 위로 쌓음.
  (기존 옹벽선 v2가 단별 분리돼 있어 구조 일치.)
- **블록 객체 = 블록정의 1개 + BlockReference 참조 삽입** (성능·수량 정확성. 참조 방식이라
  나중에 제조사 실제 블록 DWG로 블록정의만 교체하면 전체 모델이 상세화되는 장점.)
- **상단 마감 = 현장타설 콘크리트 Sweep**: 캡 단면(Polyline)→`Region.CreateFromCurves`→
  옹벽선 경로 Sweep, `SweepOptionsBuilder.Align = AlignSweepEntityToPath`(경로 직교 정렬).
  Boolean(이형 블록 자르기)은 쓰지 않음 — 블록 정수 개수 보존 + 연산 안정성.
- **뒷물림(setback) = 정지설정 구배 따라감**: 층마다 수평으로 n×층높이(200mm)만큼 물려 쌓음
  (구배 n≤0.05 → 층당 최대 10mm). 정지면 기하와 정확히 일치.

## 3. 블록 규격 (인터뷰 확정 — 스샷 Capture_2026_0720_113603)
- **"원스톤 블록": 앞면 폭 460 × 깊이 500 × 높이 200 (mm)**. 평면은 뒤로 좁아지는 사다리꼴
  (곡선 대응 실제 형상). 설정 팝업에서 치수 변경 가능하게(기본값 460×500×200).
- 1차 모델링은 **단순화 형상**(앞면 460×200, 깊이 500 박스 또는 사다리꼴 평면) — LOD 350에서
  벽면에 보이는 건 앞면뿐. 상세 형상은 블록정의 교체로 추후 대응.
- 엇갈림(조적조): 층마다 반 블록(230mm) 어긋남.

## 4. 설계 시 유의 (다음 단계에서 확정할 것)
- **캡 단면 치수(미정)**: 제안 — 폭 = 블록 깊이+여유(예 600mm), **두께 ≥ 블록 높이 200mm(예 250mm)**
  → 계단 단차(층당 200mm)가 캡 밑으로 완전히 덮이도록. JACK 확인 필요.
- **곡선/코너**: 블록 앞면을 정렬선 접선 방향으로 회전 배치. 곡선에서 뒷면 벌어짐/겹침은 실제
  시공과 동일하게 허용(사다리꼴 블록의 존재 이유). 급코너 처리 규칙은 구현 시 검토.
- **필터 규칙 상세**: 블록 상면 Z ≤ 옹벽선 보간 Z(블록 중심 위치 기준) — 옹벽선이 토우까지
  내려간 구간(벽 없음)은 자동으로 0단 → 벽 끝. 구현 시 오프라인 하네스로 검증.
- **성능 가늠**: 200m×5m 벽 ≈ 블록 1만 개 → BlockReference 1만 삽입은 DWG로 무난.
  생성은 별도 Database(사이드 DB)에서 만들어 SaveAs — 현재 도면 무영향.
- **BIM 메타데이터**: 블록 참조·캡 솔리드에 XData/속성(구조물명, 규격, 단 번호) 부여 — CSV와 대응.

## 5. 진행 순서(제안)
1) Core에 그리드 생성·필터 순수로직(`WallBlocks.cs` 가칭) + 오프라인 하네스 검증(블록 개수·계단 형상)
2) Civil에 사이드 DB DWG 생성(블록정의·참조 삽입·캡 Sweep) + DHINFRA 연결
3) CSV 물량 + 검증(InfraWorks 렌더 확인) 후 옹벽선_*.shp 제거
